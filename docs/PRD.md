# 마이링크 (Mylink) PRD (기능 정의서)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (Mylink)
- **목적**: 사용자의 다양한 링크(SNS 채널, 포트폴리오, 쇼핑몰, 블로그 등)를 하나의 페이지로 통합하여 제공하는 멀티 링크 허브 플랫폼을 구축합니다. 퍼스널 브랜딩을 강화하고 방문자 트래픽을 한 곳으로 모아 마케팅 효율성을 극대화합니다.
- **대상 사용자**: 
  - 본인만의 포트폴리오, 깃허브, 기술 블로그 등을 홍보하고 관리해야 하는 **개발자**

---

## 2. 필수 기능 목록 (MVP)

해당 기능들은 서비스 론칭을 위해 반드시 구현되어야 하는 최우선 기능입니다.

1. **계정 및 접근 관리**
   - **회원가입/로그인**: Firebase Authentication을 이용한 구글(Google) 소셜 로그인 전용 시스템
   - **고유 URL 생성**: 가입 후 설정한 고유 ID 기반으로 사용자 전용 퍼블릭 페이지 URL 활성화 (예: `mylink.com/username` 형식, `@` 프리픽스 제거)
2. **프로필 블록 관리**
   - **정보 표시**: 상단 프로필 이미지 업로드, 닉네임, 한 줄 소개(Bio) 입력 및 렌더링
3. **링크 관리 시스템 (CRUD)**
   - **링크 추가/수정/삭제**: 링크 타이틀과 대상 URL 쌍으로 일련의 링크 리스트를 생성 및 불필요한 항목 삭제 기능
   - **파비콘(Favicon) 연동**: 링크 추가 시, 등록된 목적지 대상 URL의 파비콘(Favicon) 이미지를 자동으로 로드하여 화면상의 버튼 아이콘으로 렌더링
4. **퍼블릭 프로필 페이지 뷰**
   - 대상 방문자가 접근했을 때 보여지는 최적화된 뷰 (Mobile First 접근 필요)
   - 모바일, 태블릿, 데스크탑을 모두 포괄하는 반응형(Responsive) 레이아웃

## 3. 선택 기능 목록 (고도화 및 추가 개선 제안)

MVP 이후 또는 기획에 따라 추가할 수 있는 기능입니다.

1. **추가 소셜 인증 기반 접근 (Social Auth 확장)**
   - 개발자 타겟에 맞추어 향후 깃허브(GitHub) 등 기타 소셜 로그인 추가 연동 지원
2. **테마 & 디자인 시스템 (디자인 커스터마이징)**
   - **shadcn/ui 도입 (향후)**: 초기 구현 이후 shadcn/ui를 활용하여 본격적인 디자인 시스템 및 컴포넌트 고도화
   - **테마 프리셋**: 미리 정의된 색상 팔레트와 폰트 조합 중 선택
   - **커스텀 백그라운드**: 배경 색상을 넘어 배경 이미지 또는 단색/그라데이션 직접 지정
3. **다양한 링크 위젯/블록 타입 확장**
   - **소셜 아이콘 전용 링커**: 본문 버튼으로 공간을 차지하지 않고 SNS 로고만 깔끔히 나타내는 영역
   - **콘텐츠 블록**: 유튜브 동영상 임베드, 이미지 갤러리 프리뷰
   - **텍스트 헤더**: 링크들을 카테고리별로 묶을 수 있는 텍스트 구분선(Divider) 역할
4. **통계 및 애널리틱스 (Analytics Dashboard)**
   - 내 프로필 페이지 총 방문자 수 (일/주/월간 단위 그래프)
   - 개별 링크 클릭 횟수 및 CTR (클릭 전환율)
5. **SEO 최적화 설정**
   - 나의 페이지의 Open Graph(OG 파라미터 - 소셜 공유 시 썸네일/설명) 커스텀 설정

## 4. 기술 스택 및 향후 과제

- **인증 메인**: Firebase 기반 Google 소셜 로그인 적용
- **UI 컴포넌트**: 초기 레이아웃 구조 세팅 후, **shadcn/ui**를 전면 도입하여 디자인 시스템 구축 예정
- **통계 및 애널리틱스**: 핵심 MVP 기능 우선 개발 후, 방문자 및 클릭수 통계 분석 기능은 추후 개발(선택 기능)로 조율

## 5. 파이어베이스(Firestore) 데이터 구조

Firebase Firestore를 활용한 주요 데이터베이스 스키마(Schema) 구조입니다.

### 5.1. `users` 컬렉션 (사용자 계정 정보)
사용자의 기본 계정 데이터와 퍼블릭 페이지 상단 프로필 렌더링을 위한 정보를 저장합니다.
- **Document ID**: `uid` (Firebase Auth 식별자)
  - `uid` (string): 접속을 위한 고유 사용자 ID (`mylink.com/uid`) - *가입 시 중복 검사*
  - `email` (string): 사용자 이메일 주소
  - `username` (string): 사용자의 실제 이름 (기본값: 구글 계정 이름)
  - `displayName` (string): 사용자 화면에 표시할 닉네임/핸들 (기본값: 사용자 이메일의 '@' 앞부분)
  - `bio` (string): 프로필 하단에 표시될 한 줄 소개
  - `photoUrl` (string): 업로드된 프로필 이미지의 Storage URL (또는 소셜 프로필)
  - `createdAt` (timestamp): 계정 생성 시간

### 5.2. `users/{uid}/links` 서브 컬렉션 (사용자별 등록 링크 블록)
사용자별로 등록된 개별 링크 데이터들을 저장하는 서브 컬렉션입니다.
- **Document ID**: `Auto-generated Number ID` (자동 생성 숫자 식별항목)
  - `id` (number): 자동 생성되는 링크의 고유 숫자 ID
  - `title` (string): 링크명 (타이틀, ex: "개인 기술 블로그")
  - `url` (string): 목적지 URL (ex: "https://velog.io/@alex")
  - `faviconUrl` (string): 추출된 목적지 페이지의 파비콘 이미지 주소
  - `createdAt` (timestamp): 링크 블록 최초 생성 시간
