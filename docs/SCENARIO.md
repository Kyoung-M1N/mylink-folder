# 마이링크 (Mylink) 사용자 동작 시나리오

이 문서는 마이링크 서비스의 주요 사용자 동작 흐름을 "사용자는 __하기 위해 __한다" 형식에 맞추어 포괄적으로 정의합니다.

## 1. 방문자: 마이링크 페이지 방문 및 탐색 과정
1. 방문자는 대상자(소유자)의 다양한 포트폴리오를 한눈에 탐색하기 위해, 공유받은 마이링크 주소(`mylink.com/username`)로 접속한다.
2. 방문자는 페이지 주인이 누구인지 파악하기 위해, 화면 최상단에 노출된 프로필 이미지, 닉네임, 한 줄 소개(Bio)를 확인한다.
3. 방문자는 각 링크의 목적을 명확히 인지하기 위해, 버튼 좌측에 표시된 목적지 사이트의 파비콘과 텍스트를 살핀다.
4. 방문자는 관심 있는 특정 프로젝트나 사이트 상세 내용을 보기 위해, 해당 링크 버튼을 클릭하여 외부 페이지로 이동한다.

## 2. 소유자: 회원가입 및 초기 온보딩 과정
1. 소유자는 마이링크 서비스를 시작하기 위해, 메인 랜딩 페이지에 접속하여 [구글 계정으로 로그인] 버튼을 클릭해 회원가입을 진행한다.
2. 소유자는 자신만의 고유한 접속 주소를 확보하기 위해, 회원가입 시 구글 이메일 계정의 앞부분을 기반으로 자동 생성된 고유 ID(username)를 부여받는다. (필요시 프로필 수정 메뉴에서 변경 가능)
3. 소유자는 방문자들에게 자신의 아이덴티티를 알리기 위해, 프로필 이미지(아바타)를 확인하고 닉네임과 한 줄 소개(Bio)를 작성하여 관리한다.

## 3. 소유자: 링크 추가/수정/삭제 과정
1. 소유자는 본인의 링크 포트폴리오를 관리하기 위해, 관리자 대시보드 페이지에 접속하여 구글 계정으로 로그인한다.
2. 소유자는 새로운 사이드 프로젝트나 블로그 글을 홍보하기 위해, [새 링크 추가] 버튼을 클릭하여 타이틀과 목적지 URL을 입력하고 파비콘 이미지와 함께 저장한다.
3. 소유자는 잘못 입력되거나 주소가 바뀐 링크 정보를 고치기 위해, 해당 링크 블록 우측의 [수정] 버튼을 클릭하여 내용을 변경하고 다시 저장한다.
4. 소유자는 더 이상 운영하지 않는 프로젝트 링크를 화면에서 완전히 없애기 위해, 해당 링크 블록 우측의 [삭제] 버튼을 클릭하여 영구적으로 제거한다.

## 4. 소유자: 프로필 정보 및 디자인 수정 과정
1. 소유자는 본인의 직무 역할이나 소속이 바뀌었을 때 이를 실시간으로 반영하기 위해, 대시보드 상단의 프로필 수정 영역에서 닉네임 및 한 줄 소개 텍스트를 변경하고 저장한다.
2. 소유자는 새로운 프로필 사진을 적용하기 위해, 기존의 이미지를 클릭하여 새로운 사진 파일을 업로드하고 프로필을 갱신한다.

## 5. 소유자: 마이링크 페이지 공유 및 확산 과정
1. 소유자는 본인의 링크 허브를 외부에 널리 알리기 위해, 관리자 대시보드 상단에 위치한 자신의 전용 URL(`mylink.com/username`) 우측의 [복사] 버튼을 클릭한다.
2. 소유자는 복사된 주소를 다른 사람들도 쉽게 이동할 수 있도록, 외부 서비스인 인스타그램 웹사이트란, 깃허브 메인, 이력서 문서 등에 붙여넣기 하여 등록한다.

## 6. 소유자 및 방문자: 시스템 예외 및 에러 처리 과정
1. 소유자는 링크 추가 시 목적지 URL에서 파비콘을 정상적으로 가져오지 못할 때 페이지 디자인의 혼란을 방지하기 위해, 시스템에서 제공하는 링크 기본(Default) 브라우저 아이콘이 노출되는 것을 확인한다.
2. 일반 사용자는 타인의 링크를 함부로 변경하거나 열람하지 못하도록 데이터를 보호받기 위해, 로그인되지 않은 상태에서 관리 기능이 포함된 메인 페이지에 접근 시 읽기 전용 랜딩 페이지로 렌더링되거나 로그인 안내를 받게 된다.
3. 방문자는 삭제되었거나 가입한 이력이 없는 사용자의 마이링크 주소(`mylink.com/wrong-name`)로 잘못 접속했을 경우 오류임을 인지하기 위해, 중앙에 "존재하지 않는 페이지입니다"라는 안내를 포함한 404 에러 화면을 보게 된다.
